Prototype notes

The corrected product model

The page now starts with the actual outcome: a running Bucket. It defines the two requirements—content and TikTok accounts—before introducing optional tools. Booketeer sits inside the “I need content” path because it finds and structures manuscript material; MIRA creates and stores assets, runs Buckets, publishes, and measures.

Bring-your-own is first-class

Authors can write or import on-screen text, upload images and videos, provide captions and hashtags, and connect accounts they already own. Generation, Booketeer, and managed accounts are optional and priced separately.

The corrected customer journey

Primary homepage actions open Create account. Sign in is reserved for returning authors. If a new author uses Sign in anyway, the prototype explains why no link arrived and carries the email directly into account creation.

Production security decision

An unauthenticated “no account found” response can reveal which emails use MIRA. The safest production flow is one neutral “Continue with email” action that sends either a sign-in or creation link while showing the same browser response. This prototype demonstrates Sebastian’s requested explicit state so the team can evaluate the experience.
